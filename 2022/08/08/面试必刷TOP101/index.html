反转链表

描述

给定一个单链表的头结点pHead(该头节点是有值的，比如在下图，它的val是1)，长度为n，反转该链表后，返回新链表的表头。

数据范围：0≤n≤1000

要求：空间复杂度 O(1)，时间复杂度 O(n)。

如当输入链表{1,2,3}时，

经反转后，原链表变为{3,2,1}，所以对应的输出为{3,2,1}。

解题思路：

  1. 因为链表结尾是 null，所以让 pre 的值是 null, pHead 就表示我们的头部
  2. 因为 pHead 的 next 成员马上就要指向 pre, 如果不保存 pHead 的下一个节点就会使其丢失，所以通过临时变量 t 保存它
  3. 让 pHead的 next 成员指向 pre
  4. pre 移动到 pHead 的位置，pHead 移动到 t 的位置，此时我们就回到了第一步中的情况
  5. 保持这个循环不变式直到 pHead移动到原链表结尾我们就获得了翻转之后的链表。

链表内指定区间反转

描述

将一个节点数为 size 链表 m 位置到 n 位置之间的区间反转，要求时间复杂度 O(n)O(n)，空间复杂度 O(1)O(1)。
例如：
给出的链表为1→2→3→4→5→NULL, m=2,n=4,
返回 1→4→3→2→5→NULL.

数据范围： 链表长度 0 < size≤1000，0<m≤n≤size，链表中每个节点的值满足 ∣val∣≤1000
要求：时间复杂度 O(n) ，空间复杂度 O(n)
进阶：时间复杂度 O(n)，空间复杂度 O(1)

链表中的节点每k个一组翻转

描述

将给出的链表中的节点每 k 个一组翻转，返回翻转后的链表
如果链表中的节点数不是 k 的倍数，将最后剩下的节点保持原样
你不能更改节点中的值，只能更改节点本身。

数据范围： 0≤n≤2000 ， 1≤k≤2000 ，链表中每个元素都满足 0≤val≤1000
要求空间复杂度 O(1)，时间复杂度 O(n)
例1：
给定的链表是 1→2→3→4→5
对于 k = 2 , 你应该返回 2→1→4→3→5

思路

有很多组个k需要反转，例如例1，有3组k个值需要反转

设置每组头节点为a，第k个为b

那我们不如设置一个for循环，从每组的a开始，到第k个节点结束，完成每一个区间的分别反转。这样有很多个分开的 被反转的组了。他们是单独的，但我们需要链表连接到一起，所以还有一个连接的过程。所以a.next连接到下一组反转之后的头结点中。至此，完成反转。形成一条反转并连接成功的链表。

那么在反转时需要每次调用反转方法：

设置3个指针，pre指针初始为null,cur与nxt初始等于a（每一组的头节点）

合并两个排序的链表

描述

输入两个递增的链表，单个链表的长度为n，合并这两个链表并使新链表中的节点仍然是递增排序的。

数据范围： 0≤n≤1000，−1000≤节点值≤1000
要求：空间复杂度 O(1)，时间复杂度 O(n)

如输入{1,3,5},{2,4,6}时，合并后的链表为{1,2,3,4,5,6}，所以对应的输出为{1,2,3,4,5,6}

合并k个已排序的链表

描述

合并 k 个升序的链表并将结果作为一个升序的链表返回其头节点。

数据范围：节点总数满足 0≤n≤105，链表个数满足 1≤k≤105 ，每个链表的长度满足 1≤len≤200 ，每个节点的值满足 |val| <= 1000

要求：时间复杂度 O(nlogk)

思路

将数组分成两两一组，然后调用合并函数进行合并。

最开始我是想得是，让每一个数组得第一个值与最后一个值合并，让后再让数组长度减一，最后得到合并后的结果，但是这样做算法复杂度过大，超时了。

最后，通过从中间进行分割，分别让前面的与后面的合并，然后让数组长度减半，知道数组剩一个为止，这样做的效果大大提升。

判断链表中是否有环

描述

判断给定的链表中是否有环。如果有环则返回true，否则返回false。

数据范围：链表长度 0≤n≤10000，链表中任意节点的值满足 0∣val∣<=100000

要求：空间复杂度 O(1)，时间复杂度 O(n)

输入分为两部分，第一部分为链表，第二部分代表是否有环，然后将组成的head头结点传入到函数里面。-1代表无环，其它的数字代表有环，这些参数解释仅仅是为了方便读者自测调试。实际在编程时读入的是链表的头节点。

思路

设置快慢节点，快节点步长为2，满节点步长为1，如果，有环，则快节点一定能与满节点相遇
